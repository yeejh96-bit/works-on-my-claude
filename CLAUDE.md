# 작업 규칙 (모든 프로젝트 공통 · 불변)

이 파일은 매 세션 자동으로 로드된다. 아래 규칙은 프로젝트와 상관없이 항상 적용된다.

## 나에 대해
- 나는 코딩을 모른다. 전문 용어를 최소화하고, 필요한 건 풀어서 설명한다.

## 코드 제공 방식
- 코드 수정은 내가 직접 한다. 너는 복사·붙여넣기 할 필요 없다. (바뀐 부분만 정확히 고친다)
- 보여줄 때는 무엇을 왜 바꿨는지 한국어로 설명하고, 필요한 만큼만 보여준다. (작은 파일이면 전체, 큰 파일이면 바뀐 부분 위주 — 토큰 낭비를 막는다)
- 어떤 파일인지 파일 경로를 함께 적는다.
- 파일을 새로 만들거나 바꾸면, 그다음 무엇을 하면 되는지 한 줄로 알려준다.

## 설명 방식
- 모든 설명은 한국어로 한다.

## 컨텍스트·토큰 아끼기 (하네스 습관)
- 여러 파일을 뒤져야 하는 무거운 조사는 `explore` 서브에이전트에게 맡긴다. 조사는 별도 컨텍스트에서 끝나고 결론만 돌아오므로, 메인 대화에 파일 내용이 쌓이지 않는다.
- 동작 검증은 `verify` 서브에이전트에게 맡긴다. 테스트·실행 로그가 메인 대화에 쌓이지 않고 통과/실패만 돌아온다.
- 한 번에 한 기능만 진행한다. 큰 작업을 한꺼번에 벌이지 않는다.
- 한 기능이 끝나면 새 세션으로 시작하길 사용자에게 권한다(진행상태는 PLAN.md/TASKS.md 에 남는다).
  작업 중간에 대화가 무거워지면 사용자에게 `/compact` 를 제안한다. (`/compact` 는 사용자 명령이라 네가 직접 실행할 수 없다.)
